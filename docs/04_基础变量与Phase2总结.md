# 04 基础变量与 Phase 2 总结

更新日期：2026-06-05

## 1. Phase 2 目标

Phase 2 负责建设基础数据层，确保后续衍生变量具备完整、丰富、可追溯的数据来源。

核心目标：

1. 全量重新拉取并建设基础库。
2. 保留全 A 股市场和退市股票历史数据。
3. 北交所同等维护。
4. 历史起点从 2006 年开始。
5. 建设财务报表、财务指标、财务事件、概念、行业、指数、资金流、公司行为等基础数据。
6. 将高价值原始事件从 `financial_event_raw` 拆为结构化视图或表。
7. 基础变量注册到统一数据字典。

## 2. Phase 2 已确认边界

| 主题 | 结论 |
|---|---|
| 股票范围 | 全 A 股，含沪深京、北交所、退市股票历史数据 |
| 历史区间 | 2006 年开始，初建阶段允许长时间全量拉取 |
| 数据库位置 | `data/duckdb/stock_data.duckdb` |

| 调用策略 | 支持限流、重试、断点续跑 |
| P0/P1 优先级 | 先 P0 核心基础表，再 P1 事件和增强数据 |

| 指数 | 增加中证全指、国证 A 指等，维护申万行业指数 |
| 指数成分 | 按月维护成分权重 |

## 3. 基础数据范围

| 数据域 | 主要对象 | 说明 |
|---|---|---|
| 证券主数据 | 股票基础信息、上市状态、退市状态 | 支撑全 A、退市和北交所覆盖 |
| 交易日历 | A 股交易日、前后交易日 | 所有增量和日频 as-of 的时间基准 |
| 日行情 | 个股 OHLCV、指数行情 | 原始交易事实来源 |
| 复权因子 | 个股复权因子 | 支撑 hfq/qfq 价格生成 |
| 每日基础指标 | 估值、市值、股本、换手、量比 | `daily_basic` 是一等基础表 |
| 涨跌停 | 每日涨跌停价格 | 支撑交易约束变量 |
| 财务报表 | 利润表、资产负债表、现金流量表 | raw wide + payload + 标准化字段 |
| 财务指标 | Tushare 财务指标 | 支撑 as-of、quality、growth |
| 财务事件 | 分红、预告、快报、审计、主营构成、股东户数、十大股东、回购等 | 高价值事件进入结构化入口 |
| 资金交易 | 个股资金流、两融、北向、龙虎榜 | optional 域，缺失不阻塞主流程 |
| 行业概念 | 申万行业、概念板块、成分 | 支撑板块上下文和多概念暴露 |
| 指数 | 主要指数基础信息、行情、成分权重 | 支撑市场上下文 |

## 6. 概念和行业基础设计

行业和概念设计遵守以下原则：

1. 申万一级、二级行业在第一阶段一起做，三级行业后续补齐。
2. 概念成员接受静态暴露处理。
3. 个股可以属于多个概念，设计上保留多概念列表，不只保留单一最强概念。
4. 概念强弱不能简单按成交额定义，否则宽泛概念容易长期占优。
5. 概念列表需要区分领涨和领跌，可按收益排序分别保留正向和负向列表。
6. 多周期概念列表已经增强，支持更丰富的统计窗口。

## 7. 基础变量数据字典

基础变量和衍生变量统一进入：

`outputs/variable_dictionary/global_variable_dictionary.xlsx`

基础变量 sheet 要求：

1. 中文名列使用中文叙述，不只翻译字段名。
2. 来源字段、来源接口、频率、主键、单位尽量明确。
3. 对 Tushare 字段应参考官方字段说明和实际落库字段。
4. 数据字典是人工理解全局设计的主入口，必须随字段变化维护。
