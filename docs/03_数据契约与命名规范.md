# 03 数据契约与命名规范

更新日期：2026-06-05

本文件替代历史 `data_contract.md` 的主入口。历史文件存在编码和版本漂移问题，已归档到 `docs/99_archive/20260605_full_snapshot/`。

## 1. 数据范围

默认维护全 A 股市场：

1. 上交所主板。
2. 科创板。
3. 深交所主板。
4. 创业板。
5. 北交所。
6. 退市股票历史行情和财务。

执行口径：

| 项目 | 口径 |
|---|---|
| 市场 | SSE、SZSE、BSE |
| 股票状态 | `list_status=L/D/P` 均进入证券主数据 |
| 日常更新 | 默认追踪上市股票 |
| 历史库 | 保留退市股票历史行情、复权、财务和事件 |
| 历史起点 | 默认 2006-01-01，核心交易日实际从 2006-01-04 起 |

## 2. 主键与时间字段

| 字段 | 说明 |
|---|---|
| `ts_code` | 股票代码，格式如 `600519.SH` |
| `trade_date` | 交易日，日频衍生变量统一主时间键 |
| `cal_date` | 交易日历日期 |
| `ann_date` | 公告日期，低频事件 point-in-time 可得日 |
| `end_date` | 报告期或统计截止日 |
| `period` | 财务报告期，通常与 `end_date` 同义但保留源口径 |
| `effective_date` | 信息可得或生效日期 |
| `updated_at` | 本地更新时间 |

日频主键原则：

1. 个股日频表优先使用 `(ts_code, trade_date)`。
2. 财务和事件 raw 表优先保留源接口主键，并补充 `ann_date`、`end_date`。
3. 日频 as-of 衍生表统一映射到 `(ts_code, trade_date)`。
4. 多成员关系表保留成分或事件维度，不强行压成单行。

## 3. 命名规则

| 规则 | 说明 |
|---|---|
| 字段命名 | 英文 `snake_case` |
| 表命名 | 基础表按来源域命名，衍生表统一 `derived_*` |
| 布尔字段 | 使用 `_flag` 或 `has_*` |
| 数量字段 | 使用 `_count` |
| 日期差字段 | 使用 `days_since_*`、`days_to_*` 或 `*_staleness_days` |
| 比例字段 | 使用 `_ratio`、`_rate`、`_pct`，在字段说明中声明分子、分母和单位 |
| 排名字段 | 使用 `_rank` |
| 分位字段 | 使用 `_pctile` 或 `_percentile` |
| 标准化字段 | 使用 `_z` |
| 状态字段 | 使用 `_state`，枚举值采用英文编码 |
| 口径字段 | 必须在数据字典中说明 `price_basis`、`frequency`、`source` |

禁止规则：

1. 不把事实变量命名为 `score`。
2. 不在本工程生成 `buy_signal`、`sell_signal`、`target_return` 等策略或标签字段。
3. 不用中文物理字段名，中文说明进入 Excel 数据字典。

## 4. 复权口径

| 变量类型 | 口径 |
|---|---|
| 连续价格、收益、动量、回撤 | 后复权口径 |
| 历史波动、ATR、Beta、路径风险 | 后复权口径 |
| 涨跌停、触板、一字板、停牌 | 原始交易事实口径 |
| 成交量、成交额、换手率、量比 | 原始交易事实口径 |
| 市值、估值、股本 | 原始口径或 Tushare `daily_basic` 口径 |
| 财务变量 | point-in-time as-of 口径，不复权 |
| 公司行为、治理事件 | 事件事实口径，不复权 |
| 截面转换 | 继承上游变量口径 |

补充说明：

1. 后复权适合连续历史统计，但不适合真实交易价格比较。
2. 涨跌停必须使用 raw 价格和涨跌停价，不使用复权价。
3. 与财务区间、市值、估值交叉的变量一般不复权。
4. 前复权主要用于展示和兼容，不进入核心物理表。

## 5. 涨跌停与最小价格变动单位

A 股股票第一阶段统一使用：

```text
price_tick = 0.01
```

涨跌停判断：

| 场景 | 公式 |
|---|---|
| 收盘涨停 | `close_raw >= up_limit - price_tick / 2` |
| 收盘跌停 | `close_raw <= down_limit + price_tick / 2` |
| 盘中触及涨停 | `high_raw >= up_limit - price_tick / 2` |
| 盘中触及跌停 | `low_raw <= down_limit + price_tick / 2` |
| 开盘涨停 | `open_raw >= up_limit - price_tick / 2` |
| 开盘跌停 | `open_raw <= down_limit + price_tick / 2` |
| 一字涨停 | `open/high/low/close` 均在 `up_limit ± price_tick / 2` 范围内 |
| 一字跌停 | `open/high/low/close` 均在 `down_limit ± price_tick / 2` 范围内 |

若后续扩展基金、可转债或其他品种，应增加 `price_tick` 映射表。

## 6. Point-in-time 原则

低频数据必须遵守信息可得日原则。

| 数据 | PIT 口径 |
|---|---|
| 财务报表 | 使用公告日或有效可得日映射到交易日 |
| 财务指标 | 使用公告日 as-of，不按报告期提前使用 |
| 业绩预告/快报 | 使用公告日 |
| 分红送股 | 区分公告日、股权登记日、除权除息日 |
| 审计意见 | 使用公告日 |
| 主营构成 | 优先使用报告公告日；源数据缺公告日时，使用 `end_date + 120 天` 作为保守可得日兜底，并在字段说明中登记为合成 PIT 假设 |
| 股东户数 | 使用公告日或源数据披露日 |
| 十大股东 | 使用公告日 |
| 质押统计 | `pledge_stat` 使用 `end_date` 作为 as-of 有效日，是已确认例外 |
| 未来解禁 | 只允许使用已公告的未来事件 |

禁止：

1. 不得把报告期末数据在公告日前映射给交易日。
2. 不得把未来才可知道的行业、成分、事件提前使用。
3. 不得在本工程生成未来收益标签。

## 7. 比率特殊值规则

比率字段需要区分真实数值和不可计算状态。用户确认过的方向是使用以 `-9` 开头的特殊编码区分分子/分母的零、空、负状态。

本工程当前统一原则：

1. 正常可计算时输出真实比率。
2. 分子、分母存在异常时输出特殊值，并在字段说明中记录规则。
3. 特殊值需要区分以下状态：分子为 0、分子为空、分子为负、分母为 0、分母为空、分母为负。
4. 后续 Phase 4 应将该规则实现为统一函数，避免各模块重复实现。

建议编码形式：

```text
-9ABCDEF
```

其中：

| 位 | 含义 |
|---|---|
| A | 分子是否为 0 |
| B | 分子是否为空 |
| C | 分子是否为负 |
| D | 分母是否为 0 |
| E | 分母是否为空 |
| F | 分母是否为负 |

示例：

```text
-9010100
```

表示特殊值；后六位按上述规则解释。该规则需要在 Phase 4 正式 UTF-8 数据契约和代码公共函数中固化。

## 8. 事实层原则

本工程变量层不生成：

1. 主观评分。
2. 选股结论。
3. 买卖信号。
4. 未来收益标签。

允许生成：

1. 事实状态枚举。
2. 布尔条件字段。
3. 明确条件的计数字段。
4. 透明暴露字段。
5. 截面 rank、percentile、z-score、中性化残差。

`composite_state` 已删除旧 `value_quality_score` 字段。

## 9. 文档和字典契约

任何字段变更都必须同步：

1. 数据库对象。
2. schema registry。
3. Excel 数据字典。
4. 主文档相关章节。
5. 审计报告或执行日志。

Excel 数据字典以中文字段说明为主，衍生变量需要写明公式形式的来源和加工逻辑。
