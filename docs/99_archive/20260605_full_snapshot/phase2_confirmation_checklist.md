# Phase 2 基础库建设前确认清单

生成日期：2026-05-27  
目标：进入 Phase 2 前，请确认以下事项。Phase 2 将开始建设基础数据层，涉及真实 API 拉取、DuckDB 建库、历史数据初建和质量审计。

## 1. 股票范围

当前 Phase 1 默认：

| 项目 | 默认方案 |
|---|---|
| 交易所 | SSE、SZSE、BSE |
| 市场 | 主板、创业板、科创板、北交所 |
| 股票状态 | 上市 L、退市 D、暂停 P 均保留 |
| 日常更新 | 默认只追踪上市 L |
| 历史库 | 保留退市和暂停上市历史 |

请确认：

1. 是否按该范围引入 A 股全部市场？
2. 退市股票是否在基础库初建阶段拉取历史行情和财务？
3. 北交所是否与沪深市场同等维护？

## 2. 历史区间

建议默认：

| 数据域 | 建议起点 |
|---|---|
| 交易日历 | 1990-01-01 或 Tushare 可得最早 |
| 股票日线 | 2000-01-01 起，若可得则尽量更早 |
| daily_basic | Tushare 可得最早 |
| 复权因子 | 与日线一致 |
| 指数行情 | 指数可得最早 |
| 财务报表 | 1990 年以来可得数据，或 Tushare 可得最早 |
| 资金流 | Tushare 可得最早 |
| 两融 | 2010 年以来或可得最早 |
| 北向 | 沪深港通可得最早 |

请确认：

1. 历史行情是否从 2000 年开始，还是继续沿用原项目 2006 年左右起点？
2. 财务报表是否尽量拉取 Tushare 可得最早历史？
3. 初建阶段是否允许长时间全量拉取？

## 3. 数据库与存储位置

建议默认：

| 项目 | 默认路径 |
|---|---|
| DuckDB | `data/duckdb/stock_data.duckdb` |
| Parquet | `data/parquet/` |
| 快照 | `data/snapshots/` |
| 日志 | `logs/` |
| 报告 | `reports/` |

请确认：

1. 是否使用上述新目录结构？
2. 是否需要从老项目 DuckDB 迁移部分历史数据以减少 API 拉取？
3. 是否接受 raw/detail 财务表使用 `payload_json` 保存全量原始字段？

## 4. Tushare Token 与调用策略

Phase 1 已验证老项目 token 可用，但 Phase 2 不会把 token 写入代码。

请确认：

1. 是否复用老项目 `.env` 中的 `TUSHARE_TOKEN`？
2. 是否在新项目单独创建 `.env`？
3. 初建阶段是否允许长时间运行并自动限流？
4. 若接口达到权限或频率限制，是否允许断点续跑？

## 5. P0/P1 表优先级

建议 Phase 2 分批建设：

| 批次 | 表 |
|---|---|
| Batch 1 | `trade_calendar`、`stock_basic_info`、`stock_status_history` |
| Batch 2 | `stock_daily`、`stock_daily_basic`、`stock_adj_factor`、`stock_limit_price` |
| Batch 3 | `index_basic_info`、`index_daily`、`index_weight` |
| Batch 4 | `financial_income_raw`、`financial_balance_raw`、`financial_cashflow_raw`、`financial_indicator_raw` |
| Batch 5 | 财务标准化视图、财务增强事件表 |
| Batch 6 | 行业、概念、资金流、两融、北向、龙虎榜 |
| Batch 7 | 元数据、审计、质量报告 |

请确认：

1. 是否按以上批次推进？
2. 是否优先从 P0 表开始，暂缓 P1 中的事件稀疏表？
3. 财务增强事件表是否在 Phase 2 就开始建设？

## 6. 财务报表处理

当前设计：

1. 三大报表 raw 表保留全量原始 payload。
2. 标准化视图抽取稳定字段。
3. 所有财务变量按公告日或首次公告日 point-in-time 映射。

请确认：

1. 是否接受 raw 表 `payload_json` 方案？
2. 是否需要额外展开 raw 字段为宽表物理列？
3. 是否优先建设标准化核心字段，再逐步扩展详细字段视图？

## 7. 指数和成分

默认指数池：

`000001.SH`、`000016.SH`、`000688.SH`、`000300.SH`、`000905.SH`、`000852.SH`、`399001.SZ`、`399006.SZ`

请确认：

1. 是否需要新增中证全指、万得全 A 替代指数、国证 A 指等？
2. 是否需要维护申万行业指数日线？
3. 指数成分权重是否按月/按可得日期全量保存？

## 8. Phase 2 验收标准

建议验收：

1. P0 基础表建表完成。
2. P0 数据可按选定历史范围完成初建或断点续建。
3. 每张表输出行数、日期范围、字段完整性报告。
4. 核心表主键无重复。
5. 日线、daily_basic、复权因子和涨跌停价格在最新交易日 key 对齐。
6. 指数行情和指数权重可连接。
7. 财务 raw 表能保留详细 payload，标准化视图可查询核心字段。
8. 生成 `reports/phase2_base_build_summary.md`。

## 9. 需要你明确回复的内容

请重点确认：

1. 历史起点：2000、2006，还是 Tushare 可得最早？
2. 是否迁移老库数据作为初始种子？
3. 是否接受财务 raw 表 `payload_json` 方案？
4. Phase 2 是否先做 P0，还是 P1 财务增强和概念/资金也同步做？
5. 默认指数池是否需要增删？
