# Phase 3 财务成长变量混合存储设计

生成日期：2026-06-01

## 设计结论

`derived_financial_growth` 第二阶段采用混合存储：

1. 核心物理表：`derived_financial_growth`
2. 完整计算视图：`derived_financial_growth_full_v`
3. 可选物化子集：后续按使用频率从完整视图中拆分

这个方案保留完整变量设计，又避免把全部 1,196 列长期物理化。

## 核心物理表字段范围

核心物理表保留以下字段组：

| 字段组 | 说明 |
|---|---|
| 报告期序列 | 当前报告期、前 1/2/4/8 个报告期、1/2/3 年同期报告期 |
| Tushare 成长字段 | `fina_indicator` 已提供的收入、利润、现金流、EPS、ROE、资产、权益等同比/环比 |
| 核心金额增长 | 收入、总收入、归母净利润、净利润、扣非利润、经营现金流、自由现金流、总资产、归母权益、有息债务、净债务、研发费用 |
| 核心单季倒推 | 收入、总收入、归母净利润、净利润、扣非利润、经营现金流、自由现金流、研发费用 |
| 核心财务质量变化 | ROE、毛利率、净利率、资产负债率、经营现金流/净利润、商誉/资产、应收/收入、存货/收入、费用/收入、杜邦 ROE |
| 事实型状态 | 正增长、同向增长、同比改善、连续负利润/负现金流、高商誉、高杠杆等事实标签 |

核心物理表不包含评分、综合评价或主观排序字段。

## 完整视图字段范围

完整视图 `derived_financial_growth_full_v` 覆盖第二阶段全部字段：

1. 所有金额类多周期同环比、跨年度同比和 CAGR。
2. 所有累计报表倒推单季值、单季同比、单季环比。
3. `derived_financial_quality` 中所有数值型字段的差值变化和比率变化。
4. 所有事实型状态字段。

完整视图按需计算，不承担日常增量落库职责。

## 特殊值规则

所有由分子和分母构造的比率、增长率、变化率字段继续使用统一特殊值：

```text
-9ABCDEF
```

含义见：

```text
docs/ratio_special_value_policy.md
```

## 运行建议

1. 日常任务只更新 `derived_financial_growth`。
2. 研究或审计需要全字段时查询 `derived_financial_growth_full_v`。
3. 如果某些完整视图字段被频繁查询，建立独立物化子表，例如 `derived_financial_growth_quality_change_ext`。
4. 不建议恢复 1,196 列的单一物理宽表，除非后续有明确的外部消费方要求全字段本地快照。
