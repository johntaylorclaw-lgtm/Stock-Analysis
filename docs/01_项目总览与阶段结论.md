# 01 项目总览与阶段结论

更新日期：2026-06-05

## 1. 项目定位

本工程是 A 股股票数据维护工程，目标是建设可复现、可审计、可持续更新的股票基础数据和衍生变量库。

本工程专注于“数据维护”和“事实变量构建”，不负责量化选股、回测训练、未来收益标签、买卖信号和主观评分。下游工程可以基于本工程维护的数据进行选股、建模、训练和回测。

当前主库：

`data/duckdb/stock_data.duckdb`

当前主数据字典：

`outputs/variable_dictionary/global_variable_dictionary.xlsx`

## 2. 重建原因

本项目来自对原项目 `D:\Opencode Workspace\stock_data_maintainance` 的重建。重建原因包括：

1. 原项目基础变量和衍生变量经过多轮补丁式扩展，变量体系不够系统，基础变量、衍生变量、修复字段和实验字段边界不清。
2. 原项目财务明细、概念板块、指数成分、公司行为、持有人治理等高价值数据域不够充分。
3. 原项目存在较多冗余脚本和历史修复代码，代码架构难以稳定维护。
4. 原项目日增量加载较慢，尤其是源数据修复和衍生变量刷新窗口不够精细。
5. 原项目文档和实际状态存在漂移，变量字典、README、旧模块说明和真实数据库状态并不总是一致。

重建不是简单迁移旧项目，而是在继承有效经验的基础上重新建立工程框架。

## 3. 原项目必须继承的优点

| 原项目优点 | 新项目继承方式 |
|---|---|
| DuckDB 单文件本地 OLAP 适合 A 股日频和宽变量扫描 | 继续使用 `data/duckdb/stock_data.duckdb` 作为主库 |
| 原始行情与复权因子分离 | 继续保留 raw price、adj factor、hfq/qfq 视图口径 |
| `daily_basic` 是估值、市值、股本、换手率的重要基础来源 | 升级为一等基础变量域，不作为临时修复表 |
| 模块化物理表比单个巨型宽表更易维护 | Phase 3 采用领域驱动核心物理表 |
| 统一出口视图思想有价值 | Phase 4 建设 `stock_features_core/plus/full` |
| 有序增量流水线优于无序并发更新 | Phase 4 继续建设依赖图和增量调度 |
| 日常轻量模式和历史修复模式分离合理 | 近 10 个交易日默认修复，远期修复显式确认 |
| 变量标签、完整性审计、常量列审查有价值 | 统一进入 schema registry、Excel 字典和审计报告 |
| Parquet 导出适合下游训练和分析 | 作为 Phase 4/后续工程化出口保留 |

## 4. 已确认工程边界

| 事项 | 结论 |
|---|---|
| 数据范围 | 全 A 股市场，含沪深京、北交所、退市股票历史数据 |
| 历史起点 | 2006 年左右，核心日频实际覆盖 2006-01-04 起 |
| 基础库建设 | Phase 2 全量重新拉取，不从老库迁移作为种子 |
| 财务数据 | 数据尽量丰富详实，三大报表、财务指标和财务事件均需维护 |
| 概念板块 | 静态概念成员可接受；多概念暴露需要保留列表，不只保留“最强概念” |
| 指数 | 引入主要宽基、申万行业指数和指数成分权重，成分权重按月维护 |
| 复权 | 连续历史统计优先后复权；交易事实、估值、市值、财务和事件不复权 |
| 标签/评分 | 不纳入本工程；`composite_state` 不保留 `score` 字段 |
| 历史修复 | 近 10 个交易日默认可修复；超过 10 个交易日需显式确认 |
| 运行环境 | 优先使用 WSL 环境 |

## 7. 当前核心成果

1. 全 A 股日频主干已建立。
2. 基础变量和衍生变量已注册到统一 schema 和变量字典。

4. 核心物理表和完整视图采用分层设计，兼顾查询效率和变量丰富度。
5. 财务变量采用 point-in-time as-of 映射，不提前使用报告期数据。
6. 交易行情、技术分析、估值、财务、资金流、行业概念、指数上下文、截面转换、公司行为、持有人治理、综合事实状态均已覆盖。

8. Excel 数据字典作为当前主入口维护。
