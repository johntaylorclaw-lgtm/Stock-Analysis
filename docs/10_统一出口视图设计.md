# 10 统一出口视图设计

更新日期：2026-06-07

本文定义 Phase 4 的统一特征读取入口：

1. `stock_features_core`
2. `stock_features_plus`
3. `stock_features_full`

这三个对象是下游工程读取本项目数据的优先入口。它们不是替代所有底表和完整视图，而是提供稳定、可审计、列数可控的统一出口。

## 1. 设计目标

统一出口视图解决三个问题：

1. 下游不需要手工连接十多个 `derived_*` 模块表。
2. 日常读取不必扫描 2,000+ 个衍生字段。
3. 字段口径、复权、PIT、事实层原则由本工程统一维护。

设计原则：

1. `core` 面向日常模型和分析，保留高频核心事实字段。
2. `plus` 在 `core` 基础上增加公司行为、治理和综合事实状态。
3. `full` 在 `plus` 基础上补充证券主数据、原始行情口径、基础估值、市值、资金和北向基础字段。
4. 三个视图均不包含主观评分字段。
5. 三个视图均以 `(ts_code, trade_date)` 为逻辑主键。

## 2. 当前规模

| 视图 | 列数 | 用途 |
|---|---:|---|
| `stock_features_core` | 318 | 日常核心事实变量出口，覆盖日频主干、交易技术、估值、财务 as-of 与核心资金/行业/指数/截面字段 |
| `stock_features_plus` | 1,198 | 增强事实变量出口，完整纳入财务质量/成长、资金、行业概念、指数市场、公司行为、治理和综合状态字段 |
| `stock_features_full` | 1,602 | 完整事实出口，在 `plus` 基础上纳入横截面全量字段和基础 enriched 字段 |

三档视图均已注册到 `config/schema_registry.json`，并进入全局 Excel 数据字典生成范围。

Phase 5 覆盖率审计记录：

| 审计项 | 结论 |
|---|---|
| 审计报告 | `reports/phase5_feature_view_coverage_audit.md` |
| Phase 3 股票级模块有效字段 | 约 1,392 个 |
| 当前 `stock_features_full` 规模 | 1,602 列 |
| 主要结论 | `full` 已覆盖全部股票级 Phase 3 模块字段，并补充基础 enriched 字段；`plus` 已覆盖除横截面全量字段外的主要事实模块；`core` 已扩充至高频稳定核心字段 |

审计后的设计判断：

1. Phase 5 已将三档视图扩充为动态生成结构，避免 1000+ 字段依赖手工 SQL 维护。
2. `full` 接受千列级全量事实出口，用于审计和全量研究；日常下游读取仍建议优先使用 `core` 或列裁剪后的 `plus/full`。
3. 对重复字段采用模块前缀别名，避免不同模块同名字段静默覆盖。

## 3. `stock_features_core`

`stock_features_core` 从 `derived_daily_spine` 出发，左连接主要核心衍生模块。

字段组：

| 字段组 | 来源 | 示例 |
|---|---|---|
| 日频主干 | `derived_daily_spine` | raw/hfq 价格、收益、涨跌停、上市状态、质量标记 |
| 价格技术 | `derived_price_technical` | `ma_20_hfq`、`ma_60_hfq`、`rsi_14` |
| 成交流动性 | `derived_volume_liquidity` | `volume_ma_20`、`amount_ma_20`、`amihud_20` |
| 收益动量 | `derived_return_momentum` | `ret_20_hfq`、`ret_60_hfq`、`momentum_60_20_hfq` |
| 波动风险 | `derived_volatility_risk` | `hv_60`、`max_drawdown_60_hfq`、`downside_vol_60` |
| 交易约束 | `derived_trading_constraint` | `limit_up_days_20`、`consecutive_limit_up_days`、`tradable_state` |
| 估值规模 | `derived_valuation_size` | `pe_ttm`、`pb`、`total_mv`、`pe_ttm_pct_5y` |
| 财务 as-of/质量/成长 | `derived_financial_*` | `latest_report_end_date`、`roe_asof`、`revenue_yoy_asof` |
| 资金流 | `derived_capital_flow` | `main_net_amount`、`main_flow_sum_20` |
| 行业概念 | `derived_sector_concept_context` | `sw_l1_name`、`concept_names_all` |
| 指数市场 | `derived_index_market_context` | `market_up_ratio`、`primary_index_name` |
| 截面转换 | `derived_cross_sectional` | `ret_20_hfq_pct_all_desc`、各类 exposure z |

## 4. `stock_features_plus`

`stock_features_plus` 在 `core` 基础上增加低频事件、治理和状态字段。

字段组：

| 字段组 | 来源 | 示例 |
|---|---|---|
| 公司行为 | `derived_corporate_action` | `latest_corp_action_date`、`cash_dividend_ttm`、`forecast_type_latest`、`repurchase_amount_365d` |
| 持有人治理 | `derived_ownership_governance` | `pledge_ratio_asof`、`holder_num_asof`、`top10_holder_ratio_latest` |
| 综合事实状态 | `derived_composite_state` | `trend_state`、`valuation_percentile_state`、`concept_heat_state`、`pledge_ratio_state` |

注意：

1. 本视图不包含 `value_quality_score`。
2. 所有综合状态字段均为事实状态或条件枚举，不表达投资建议。

## 5. `stock_features_full`

`stock_features_full` 在 `plus` 基础上补充基础 enriched 字段。

字段组：

| 字段组 | 来源 | 示例 |
|---|---|---|
| 证券主数据 | `stock_base_daily_enriched` | `symbol`、`name`、`security_market`、`security_exchange`、`list_status` |
| 前复权展示价格 | `stock_base_daily_enriched` | `open_qfq_current`、`close_qfq_current` |
| 原始行情别名 | `stock_base_daily_enriched` | `raw_open`、`raw_close`、`raw_amount` |
| 基础交易指标 | `stock_base_daily_enriched` | `turnover_rate`、`turnover_rate_free` |
| 基础估值市值 | `stock_base_daily_enriched` | `base_pe_ttm`、`base_pb`、`total_mv`、`circ_mv` |
| 基础资金与北向 | `stock_base_daily_enriched` | `net_mf_amount`、`margin_balance`、`northbound_hold_ratio` |

命名注意：

1. 为避免字段重名，动态生成器会保留首次出现字段名，对后续同名字段加模块前缀，例如 `base_total_mv`、`cross_ret_20_hfq_pct_all_desc`。
2. `full` 是全量事实出口，但仍以 `(ts_code, trade_date)` 为股票日频粒度，不做非股票级事件表的笛卡尔式合并。

## 6. 维护规则

1. 三个视图由 `stock-maintain create-views` 创建或刷新，Phase 5 起由 `src/stock_maintainance/views.py` 动态生成扩充列集。
2. 视图字段变更必须运行 `scripts/sync_feature_view_schema_registry.py` 同步 `config/schema_registry.json`。
3. 视图字段变更后必须重新生成 `outputs/variable_dictionary/global_variable_dictionary.xlsx`。
4. 三个视图不得包含 `score` 字段。
5. 三个视图不得使用未来收益、买卖信号或策略标签。
6. 下游如需要 1000+ 列扩展变量，应直接读取对应 `derived_*_full_v`，而不是要求 `stock_features_full` 无限扩张。

## 7. 验收规则

统一出口视图的最低验收：

1. `stock-maintain create-views` 成功。
2. `stock_features_core/plus/full` 均存在。
3. 三个视图列数与 schema registry 一致。
4. 三个视图无 `score` 字段。
5. `stock-maintain validate-config` 通过。
6. `stock-maintain docs-check` 通过。
7. `python -m pytest` 通过。
