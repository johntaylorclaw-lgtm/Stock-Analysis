# 06 Phase 3 模块设计合订本

更新日期：2026-06-05

本文件合并 Phase 3 模块设计的当前事实口径。详细历史设计和执行过程可在 `docs/99_archive/20260605_full_snapshot/` 中追溯。

## 1. 日频主干

`derived_daily_spine` 是所有日频衍生变量的连接基准。

核心字段组：

| 字段组 | 内容 |
|---|---|
| 主键与状态 | `ts_code`、`trade_date`、`is_trade`、`is_listed_asof`、`list_status_asof`、`days_since_list`、`market`、`exchange` |
| raw OHLCV | `open_raw`、`high_raw`、`low_raw`、`close_raw`、`pre_close_raw`、`change_raw`、`pct_chg_raw`、`volume`、`amount`、`amplitude_raw` |
| 复权因子 | `adj_factor`、`latest_adj_factor_asof`、`has_adj_factor` |
| 后复权价格 | `open_hfq`、`high_hfq`、`low_hfq`、`close_hfq`、`pre_close_hfq` |
| 基础收益 | `ret_1_raw`、`ret_1_hfq`、`log_ret_1_hfq`、`overnight_ret_hfq`、`intraday_ret_hfq`、`gap_open_hfq` |
| 日内结构 | `high_low_range_hfq`、`close_position_hfq` |
| 涨跌停 | `up_limit`、`down_limit`、`limit_up_flag`、`limit_down_flag`、触板、开盘封板、涨跌停缺口 |
| 质量标记 | `has_price`、`has_limit_price`、`price_valid_flag`、`missing_reason` |

设计口径：

1. raw 价格用于交易事实。
2. hfq 价格用于连续历史统计。
3. qfq 价格不进入核心物理表，只在完整视图中提供展示和兼容用途。

## 2. 交易行情与技术分析

交易行情与技术分析拆分为五个核心表：

1. `derived_price_technical`
2. `derived_volume_liquidity`
3. `derived_return_momentum`
4. `derived_volatility_risk`
5. `derived_trading_constraint`

通用原则：

1. 连续价格、收益、动量、回撤使用后复权口径。
2. 成交额、成交量、换手率使用原始交易事实口径。
3. 涨跌停、触板、一字板使用 raw 价格和 `price_tick=0.01`。
4. 核心表保存常用周期。
5. 完整视图展开丰富周期。

### 2.1 价格技术

核心物理表保留均线、均线距离、均线斜率、RSI、价格区间位置等高频字段。

完整视图扩展：

| 字段组 | 周期/字段 |
|---|---|
| 均线 | `ma_{3,5,10,20,30,60,90,120,180,250}_hfq` |
| EMA | `ema_{5,10,20,60,120}_hfq` |
| 均线距离 | `close_to_ma_{5,10,20,60,120,250}_hfq` |
| 均线斜率 | `ma_{5,20,60,120}_slope_{5,20,60,120}_hfq` |
| 振荡指标 | `rsi_{6,14,24}`、`bias_{6,12,24}_hfq`、`cci_20_hfq`、`wr_14_hfq` |
| 通道位置 | `price_position_{20,60,120,250}_hfq`、布林带字段 |
| MACD/KDJ | 进入完整视图，不进入核心物理表 |

### 2.2 收益动量

核心采用后复权收益。完整视图扩展多周期收益、对数收益、近端剔除动量、上涨/下跌天数、新高新低、距离高低点。

主要周期：

```text
1, 2, 3, 5, 10, 20, 30, 60, 90, 120, 180, 250
```

### 2.3 波动风险

核心字段覆盖历史波动、区间振幅、回撤、下行风险和风险路径。年化交易日统一为 242。

完整视图扩展：

1. 多周期波动率。
2. Parkinson/Garman-Klass 等高低价波动口径。
3. 多周期最大回撤。
4. 下行波动、偏度、峰度和尾部风险。

### 2.4 成交流动性

成交量、成交额、换手率、量比、Amihud、成交冲击类字段全部使用原始交易事实口径，不做复权。

完整视图周期扩展包括：

```text
2, 3, 5, 10, 20, 30, 60, 120
```

### 2.5 交易约束

覆盖涨跌停、触板、开盘封板、一字板、连续涨跌停、停牌/缺失、价格质量和交易约束状态。

## 3. 估值与规模

核心对象：

`derived_valuation_size`

主要维护：

1. PE、PE_TTM、PB、PS、PS_TTM、股息率。
2. 总市值、流通市值、自由流通市值。
3. 总股本、流通股本、自由流通股本。
4. 估值有效性标记。
5. 估值、市值、股本历史分位。
6. 规模分组和流通性规模暴露。

设计口径：

1. 估值和市值不复权。
2. 使用 `daily_basic` 原始口径。
3. 非核心多周期分位和相对估值进入完整视图。

## 4. 财务衍生变量

核心对象：

1. `derived_financial_asof`
2. `derived_financial_quality`
3. `derived_financial_growth`

通用原则：

1. 财务变量必须 point-in-time。
2. 使用公告日或可得日映射到交易日。
3. 不将报告期数据提前使用。
4. 不生成综合质量分。
5. 比率字段遵守特殊值规则。

### 4.1 财务 as-of

维护当前交易日可得的最近财务报告、报告期、公告日、滞后天数和核心财务数值。

### 4.2 财务质量

覆盖事实型质量指标，包括：

1. 盈利能力：ROE、ROA、毛利率、净利率、营业利润率。
2. 费用结构：销售费用率、管理费用率、研发费用率、财务费用率。
3. 资产结构：货币资金、应收、存货、商誉、有息负债、流动资产和非流动资产结构。
4. 偿债能力：资产负债率、流动比率、速动比率、现金覆盖、有息负债率。
5. 营运效率：应收周转、存货周转、总资产周转、经营现金流与收入/利润匹配。
6. 现金流质量：经营现金流、自由现金流、现金利润覆盖。
7. 杜邦拆解：净利率、总资产周转、权益乘数等事实分解。

### 4.3 财务成长

采用物理核心表 + 完整视图。

设计原因：

1. 用户要求财务成长多周期同环比尽量丰富。
2. 全部物理落库会导致空间占用过大。
3. 核心字段物理化，宽周期扩展进入视图更合理。

字段范围：

1. 单季同比、环比。
2. TTM 同比、环比。
3. 多年同周期变化。
4. ROE、ROA、毛利率、净利率等质量指标的多周期变化率。
5. 收入、利润、现金流、资产负债、营运效率的多周期增长。

## 5. 资金流与交易行为

核心对象：

1. `derived_capital_flow`
2. `derived_northbound_flow_cache`
3. `derived_capital_flow_event_cache`

主要维护：

1. 大小单资金流。
2. 主力资金流。
3. 资金净流入占成交额比例。
4. 融资融券余额、买入、偿还、净变化。
5. 北向持股、持股市值、持股比例、变化量。
6. 龙虎榜和机构席位事件。

资金流金额、成交额、市值均保持事实口径，跨单位比例在字段说明中标明。

## 6. 行业、概念与指数市场上下文

核心对象：

1. `derived_sector_concept_context`
2. `derived_index_market_context`

行业概念维护：

1. 申万一级和二级行业。
2. 概念成分静态暴露。
3. 个股多概念列表。
4. 多周期概念强弱列表。
5. 行业相对收益、成交额和市场位置。

概念排序原则：

1. 不只按成交额定义“强概念”，避免宽泛概念长期占优。
2. 同时保留收益排序、成交额排序、上涨/下跌方向。
3. 对个股多概念暴露保留列表字段，避免丢失多主题属性。

指数市场维护：

1. 主要宽基指数。
2. 申万行业指数。
3. 指数成员和权重。
4. 市场宽度、上涨下跌家数、涨跌停统计。
5. 按交易日广播到个股视图。

## 7. 截面转换

核心对象：

`derived_cross_sectional`

主要维护：

1. 全市场 rank、percentile、z-score。
2. 市场板块内 rank、percentile、z-score。
3. 交易所内 rank、percentile、z-score。
4. 申万行业内 rank、percentile、z-score。
5. 市值/行业中性化残差。
6. 透明暴露字段。

注意：

1. 截面字段不命名为 `score`。
2. 字段只表达相对位置，不表达投资建议。
3. 第三阶段更复杂的截面组合留到全量底表完成后再实现。

## 8. 公司行为

核心对象：

`derived_corporate_action`

主要维护：

1. 分红送股。
2. 业绩预告。
3. 业绩快报。
4. 审计意见。
5. 主营构成。
6. 回购。
7. 解禁。

设计口径：

1. 公司行为使用原始事实口径。
2. 未来解禁窗口只统计已公告事件。
3. 分红区分公告日、股权登记日、除权除息日。
4. 稀疏事件以 timeline 视图辅助人工检查。

## 9. 持有人治理

核心对象：

`derived_ownership_governance`

主要维护：

1. 股权质押统计和明细。
2. 股东户数。
3. 十大股东。
4. 十大流通股东。
5. 股东集中度。
6. 持有人事件时间线。

已确认边界：

1. 质押比例事实标记使用 10/30/50 三档。
2. 同时纳入控股股东、实际控制人、高管相关口径中可得字段。
3. 使用原始口径，不做主观治理评分。

## 10. 综合事实状态

核心对象：

`derived_composite_state`

主要维护：

1. 股票身份和可交易状态。
2. 趋势状态。
3. 流动性和波动状态。
4. 估值和规模状态。
5. 财务披露、质量和成长状态。
6. 资金参与者状态。
7. 行业、概念、指数、市场上下文状态。
8. 公司行为和治理状态。
9. 多域条件计数。

本模块已删除旧 `value_quality_score`，不包含任何评分字段。状态枚举使用英文编码，便于下游工程稳定解析。
