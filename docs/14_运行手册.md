# 14 运行手册

更新日期：2026-06-07

## 1. 环境入口

优先使用 WSL：

```bash
cd "/mnt/d/Opencode Workspace/Stock_Maintainance"
.venv-wsl/bin/stock-maintain --help
```

## 2. 日批 daily-light

默认运行时间：北京时间 20:00。

该时间不是任意设置。2026-06-08 首日日加载审计显示，`daily_basic`、`moneyflow`、`moneyflow_hsgt`、`top_list`、`top_inst`、`index_daily` 等 T+0 接口在 15:56 尚未完整发布，但晚间可获取。因此 Hermes Agent 的默认日加载任务调整为 20:00 执行。

项目内 Hermes 调度配置记录在：

`config/hermes_agent.json`

预检查：

```bash
.venv-wsl/bin/stock-maintain daily-light \
  --as-of-date <YYYY-MM-DD> \
  --dry-run \
  --output-prefix <run_id>
```

执行：

```bash
.venv-wsl/bin/stock-maintain daily-light \
  --as-of-date <YYYY-MM-DD> \
  --output-prefix <run_id>
```

Hermes Agent 接续汇总：

```bash
.venv-wsl/bin/stock-maintain summarize-run \
  --mode daily \
  --run-id <run_id>
```

验收：

1. `daily-light` 总状态为 `pass`。
2. precheck 和 postcheck 均为 `pass`。
3. 若待增量交易日为 0，也应确认 `sync-master` 已执行。
4. 若状态为 `blocked`，不得继续补数，先确认是否超过 10 个交易日。
5. 若 `margin_detail`、`northbound_holding` 仅在最新交易日缺失，且验证报告标记为预期 T+1 延迟，不视为日批阻塞。

## 3. 日常强修复 daily-full

当日频源数据晚到、`daily-light` 因“无缺口”跳过、或最近窗口需要强制重拉时，使用 `daily-full`。

预检查：

```bash
.venv-wsl/bin/stock-maintain daily-full \
  --as-of-date <YYYY-MM-DD> \
  --reload-trade-days 1 \
  --dry-run \
  --output-prefix <run_id>
```

执行：

```bash
.venv-wsl/bin/stock-maintain daily-full \
  --as-of-date <YYYY-MM-DD> \
  --reload-trade-days 1 \
  --output-prefix <run_id>
```

若本次修复后需要同步刷新 weekly 参照快照：

```bash
.venv-wsl/bin/stock-maintain daily-full \
  --as-of-date <YYYY-MM-DD> \
  --reload-trade-days <N> \
  --refresh-weekly-snapshot \
  --output-prefix <run_id>
```

注意：`daily-full --dry-run` 会运行 `validate-daily`，真实库上可能需要一到两分钟；它不会执行 Tushare 拉取、衍生落库或视图刷新。

## 4. 周检 weekly-full

预检查：

```bash
.venv-wsl/bin/stock-maintain weekly-full \
  --as-of-date <YYYY-MM-DD> \
  --dry-run \
  --output-prefix <run_id>
```

执行：

```bash
.venv-wsl/bin/stock-maintain weekly-full \
  --as-of-date <YYYY-MM-DD> \
  --auto-create-missing-snapshot \
  --output-prefix <run_id>
```

Hermes Agent 接续汇总：

```bash
.venv-wsl/bin/stock-maintain summarize-run \
  --mode weekly \
  --run-id <run_id>
```

验收：

1. 字段级对照表数为 25。
2. 通过表为 25。
3. 键差异为 0。
4. 字段差异为 0。
5. 若实际比较窗口与请求窗口不同，应在周报中说明快照共同窗口。
6. 若结果为 `snapshot_created`，说明刚刚创建了参照快照，需要再跑一次 weekly-full 完成独立 compare。

## 5. 当前状态汇总

```bash
.venv-wsl/bin/stock-maintain summarize-run \
  --mode status \
  --as-of-date <YYYY-MM-DD>
```

输出：

`reports/summaries/status_summary_*.md`

## 6. 数据字典刷新

```bash
.venv-wsl/bin/stock-maintain refresh-dictionary \
  --output-prefix <run_id>
```

验收：

1. `config/schema_registry.json` 与统一出口实际列同步。
2. `docs/generated/*` 已刷新。
3. `outputs/variable_dictionary/global_variable_dictionary.xlsx` 已重建。
4. `docs-check` 通过。

## 8. 历史修复

规则：

1. 近 10 个交易日内可按 daily-light 规则处理。
2. 超过 10 个交易日必须显式确认。
3. 修复前生成计划，修复后运行 targeted validation 或 weekly-full。
4. 不在 Hermes Agent 中自动触发影子全量重建。

## 9. 常见故障

| 问题 | 处理 |
|---|---|
| DuckDB 文件锁 | 避免并行打开多个写/读任务，串行运行 CLI |
| WSL Node 无法加载 artifact-tool | 使用 `refresh-dictionary`，其内部调用 Windows bundled Node |
| weekly-full 请求窗口早于快照窗口 | 自动取交集；若无交集则 blocked |
| daily-light blocked | 多数为超过 10 个交易日，需人工确认 |
| docs-check stale | 运行 `refresh-dictionary` 或 `docs-generate` 后复查 |

## 10. Agent Skill

项目内 Skill 位于：

`Skill/stock-maintenance-ops/`

Hermes Agent 使用该 Skill 的规则：

1. daily-light 默认在北京时间 20:00 执行。
2. daily-light 后运行 `summarize-run --mode daily`。
3. weekly-full 后运行 `summarize-run --mode weekly`。
4. 当前状态检查运行 `summarize-run --mode status`。
5. 输出 Markdown 路径，不生成投资建议。
