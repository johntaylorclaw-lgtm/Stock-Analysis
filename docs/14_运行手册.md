# 14 运行手册

更新日期：2026-06-07

## 1. 环境入口

优先使用 WSL：

```bash
cd "/mnt/d/Opencode Workspace/Stock_Maintainance"
.venv-wsl/bin/stock-maintain --help
```

## 2. 日批 daily-light

预检查：

```bash
.venv-wsl/bin/stock-maintain daily-light \
  --as-of-date <YYYY-MM-DD> \
  --dry-run \
  --output-prefix <run_id>
```

执行：

```bash
.venv-wsl/bin/stock-maintain daily-light \
  --as-of-date <YYYY-MM-DD> \
  --output-prefix <run_id>
```

Hermes Agent 接续汇总：

```bash
.venv-wsl/bin/stock-maintain summarize-run \
  --mode daily \
  --run-id <run_id>
```

验收：

1. `daily-light` 总状态为 `pass`。
2. precheck 和 postcheck 均为 `pass`。
3. 若待增量交易日为 0，也应确认 `sync-master` 已执行。
4. 若状态为 `blocked`，不得继续补数，先确认是否超过 10 个交易日。

## 3. 周检 weekly-full

预检查：

```bash
.venv-wsl/bin/stock-maintain weekly-full \
  --as-of-date <YYYY-MM-DD> \
  --dry-run \
  --output-prefix <run_id>
```

执行：

```bash
.venv-wsl/bin/stock-maintain weekly-full \
  --as-of-date <YYYY-MM-DD> \
  --output-prefix <run_id>
```

Hermes Agent 接续汇总：

```bash
.venv-wsl/bin/stock-maintain summarize-run \
  --mode weekly \
  --run-id <run_id>
```

验收：

1. 字段级对照表数为 24。
2. 通过表为 24。
3. 键差异为 0。
4. 字段差异为 0。
5. 若实际比较窗口与请求窗口不同，应在周报中说明快照共同窗口。

## 4. 当前状态汇总

```bash
.venv-wsl/bin/stock-maintain summarize-run \
  --mode status \
  --as-of-date <YYYY-MM-DD>
```

输出：

`reports/summaries/status_summary_*.md`

## 5. 数据字典刷新

```bash
.venv-wsl/bin/stock-maintain refresh-dictionary \
  --output-prefix <run_id>
```

验收：

1. `config/schema_registry.json` 与统一出口实际列同步。
2. `docs/generated/*` 已刷新。
3. `outputs/variable_dictionary/global_variable_dictionary.xlsx` 已重建。
4. `docs-check` 通过。

## 6. 样本抽检

```bash
.venv-wsl/bin/stock-maintain sample-stock \
  --ts-code <ts_code> \
  --start-date <YYYY-MM-DD> \
  --end-date <YYYY-MM-DD> \
  --rows 5 \
  --output-prefix <run_id>
```

输出：

`outputs/phase5/*.xlsx`

## 7. 历史修复

规则：

1. 近 10 个交易日内可按 daily-light 规则处理。
2. 超过 10 个交易日必须显式确认。
3. 修复前生成计划，修复后运行 targeted validation 或 weekly-full。
4. 不在 Hermes Agent 中自动触发影子全量重建。

## 8. 常见故障

| 问题 | 处理 |
|---|---|
| DuckDB 文件锁 | 避免并行打开多个写/读任务，串行运行 CLI |
| WSL Node 无法加载 artifact-tool | 使用 `refresh-dictionary`，其内部调用 Windows bundled Node |
| weekly-full 请求窗口早于快照窗口 | 自动取交集；若无交集则 blocked |
| daily-light blocked | 多数为超过 10 个交易日，需人工确认 |
| docs-check stale | 运行 `refresh-dictionary` 或 `docs-generate` 后复查 |

## 9. Agent Skill

项目内 Skill 位于：

`Skill/stock-maintenance-ops/`

Hermes Agent 使用该 Skill 的规则：

1. daily-light 后运行 `summarize-run --mode daily`。
2. weekly-full 后运行 `summarize-run --mode weekly`。
3. 当前状态检查运行 `summarize-run --mode status`。
4. 输出 Markdown 路径，不生成投资建议。
