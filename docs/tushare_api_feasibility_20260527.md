# Tushare API 实证可行性验证

验证日期：2026-05-27  
验证方式：使用老项目 `.env` 中的 Tushare token 进行小样本 API 调用。  
安全说明：本文不记录、不展示 token。

## 1. 验证结论

当前 token 可支持新项目 Phase 1 需要的主要数据引入：

1. A 股全市场范围可覆盖深交所、上交所、北交所，含主板、创业板、科创板、北交所。
2. 股票日线、日度基础指标、复权因子、涨跌停价格可正常获取。
3. 主要指数基础信息、指数行情和指数成分权重可正常获取。
4. 申万行业分类和行业成分可正常获取。
5. Tushare 概念定义和概念成分可正常获取。
6. 财务三大报表可获取远多于原项目使用字段的详细字段：利润表 85 列、资产负债表 152 列、现金流量表 97 列。
7. 财务增强接口中，财务指标、业绩预告、审计意见、主营业务构成、股东户数、十大股东、十大流通股东、回购可获取样本数据。
8. 部分事件型接口在单只样本股票上返回 0 行，但接口和字段可访问，例如业绩快报、分红、质押、限售解禁；这类数据应按全市场事件稀疏表设计。

## 2. A 股市场范围验证

`stock_basic(exchange="", list_status="L")` 返回当前上市股票 5,524 只：

| 交易所 | 市场 | 上市数量 |
|---|---|---:|
| BSE | 北交所 | 316 |
| SSE | 主板 | 1,705 |
| SSE | 科创板 | 610 |
| SZSE | 主板 | 1,495 |
| SZSE | 创业板 | 1,398 |

退市股票 `list_status="D"` 返回 325 只：

| 交易所 | 市场 | 退市数量 |
|---|---|---:|
| BSE | 北交所 | 4 |
| SSE | 主板 | 137 |
| SSE | 科创板 | 2 |
| SZSE | 主板 | 142 |
| SZSE | 创业板 | 40 |

`list_status="P"` 当前返回 0 行。

设计结论：新项目应引入 A 股所有市场，包括上交所主板、科创板，深交所主板、创业板，北交所，并保留退市股票历史。

## 6. 财务接口验证

以 `600519.SH`、2024-01-01 至 2026-05-26 为样本：

| API | 返回行数 | 字段数 | 结论 |
|---|---:|---:|---|
| `income` | 11 | 85 | 利润表详细字段可用 |
| `balancesheet` | 11 | 152 | 资产负债表详细字段可用 |
| `cashflow` | 15 | 97 | 现金流量表详细字段可用 |
| `fina_indicator` | 11 | 108 | 财务指标详细字段可用 |
| `forecast` | 3 | 13 | 业绩预告可用 |
| `express` | 0 | 15 | 接口字段可用，样本期无数据 |
| `dividend` | 0 | 14 | 接口字段可用，样本期无数据 |
| `fina_audit` | 3 | 7 | 审计意见可用 |
| `fina_mainbz` | 40 | 8 | 主营构成可用 |
| `disclosure_date` | 0 | 5 | 接口字段可用，样本参数无数据 |
| `stk_holdernumber` | 13 | 4 | 股东户数可用 |
| `top10_holders` | 90 | 9 | 十大股东可用 |
| `top10_floatholders` | 110 | 9 | 十大流通股东可用 |
| `pledge_stat` | 0 | 7 | 接口字段可用，样本股票无数据 |
| `pledge_detail` | 0 | 14 | 接口字段可用，样本股票无数据 |
| `repurchase` | 23 | 9 | 回购数据可用 |
| `share_float` | 0 | 7 | 接口字段可用，样本股票无数据 |

设计结论：

1. 财务三大报表应采用“详细原始表 + 标准化核心视图/字段”的设计，而不是只取原项目那几十个字段。
2. 报表详细字段应尽量全量保留，后续衍生变量从标准化视图读取。
3. 事件型财务明细不能用单只股票样本是否有数据判断是否保留，应按全市场稀疏事件表设计。

## 7. 下一步约束

1. 所有 API 接入代码不得硬编码 token。
2. 接口字段应先进入 `metadata_source_api` 和 `metadata_table_schema`。
3. 对于字段多、行业差异大的三大报表，建议保留 raw/detail 表，再生成标准化视图。
4. 每个接口在 Phase 1 实现时都要加入小样本 smoke test。
