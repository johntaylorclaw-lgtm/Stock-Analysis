## 2. 当前主数据字典

Excel 数据字典是字段设计和变量说明的主要入口：

`outputs/variable_dictionary/global_variable_dictionary.xlsx`

维护要求：

1. 每个基础变量表、衍生变量表一个 sheet。
2. `Table_Index` 提供全局入口，Sheet 名配置跳转链接。
3. 基础变量以中文字段说明为主。
4. 衍生变量必须包含中文名、具体含义、衍生逻辑、来源表和来源字段。
5. 字段新增、删除、改名后必须重新生成 Excel 字典。

## 4. 归档原则

1. 主目录只保留当前事实状态文档。
2. 早期草案、阶段性确认、旧计划、旧 Markdown 字典快照全部放入 `99_archive`。
3. 后续新增文档必须编号，并说明它替代或继承了哪些旧文档。
4. 新文档不能只写结论，应保留关键边界、设计原因、对象清单、已知限制和后续动作。
5. 若主文档与数据库、schema registry、Excel 数据字典不一致，以数据库和注册表审计结果为准，并立即更新文档。

## 5. 推荐阅读顺序

1. 新成员先读 `01_项目总览与阶段结论.md`，理解工程定位和阶段结论。
2. 需要理解整体架构时读 `02_项目整体设计.md`。
3. 写 SQL、写脚本、设计变量前读 `03_数据契约与命名规范.md`。
4. 查基础变量和 Phase 2 范围读 `04_基础变量与Phase2总结.md`。
5. 查 Phase 3 已完成对象和规模读 `05_衍生变量Phase3总览.md`。
6. 查每个衍生模块设计细节读 `06_Phase3模块设计合订本.md`。
7. 运行、刷新、更新字典读 `07_运行维护与数据字典.md`。
8. 验收、审计和下一阶段安排读 `08_审计验收与遗留事项.md`。
9. 启动 Phase 4 前读 `09_Phase4启动审计与实施计划.md`。
10. 下游读取统一特征入口前读 `10_统一出口视图设计.md`。
11. 做增量性能优化和脚本迁移前读 `11_Phase4增量窗口与脚本分类.md`。
12. 做 Phase 5 验证、样本审查和交付闭环前读 `12_Phase5验证文档交付闭环设计.md`。
13. 做 Agent Skill、运行手册和自动报告汇总前读 `13_AgentSkill运行手册与自动报告设计.md`。
14. 执行 daily-light、weekly-full、Hermes Agent 报告汇总、数据字典刷新和样本抽检时读 `14_运行手册.md`。
