## 7. 遗留事项

| 遗留事项 | 影响 | 建议 |
|---|---|---|
| 统一出口视图 `stock_features_core/plus/full` 尚未实施 | 不影响 Phase 3 主体模块使用，但下游统一读取入口仍需建设 | Phase 4 第一项 |
| 增量刷新统一调度尚未完全产品化 | 当前已完成全量历史构建，日增量机制需统一封装 | Phase 4 建设任务编排和依赖图 |
| 比率特殊值规则尚未公共函数化 | 不影响已落库事实，但长期维护需要统一实现 | Phase 4 固化为公共函数和测试 |
| 申万三级行业尚未补齐 | 不影响一级/二级行业上下文，但行业层级仍可扩展 | 后续补齐 |
| 北交所、退市股等边缘样本覆盖需持续监控 | 已纳入数据范围，但持续更新中仍需质量审计 | 加入每日质量报告 |
| 报告区仍保留大量运行日志 | 不影响使用，但目录较重 | 后续按审计、运行日志、归档再分层 |

## 8. 下一阶段建议

建议进入 Phase 4：工程化运行与统一出口。

优先顺序：

1. 建设 `stock_features_core`、`stock_features_plus`、`stock_features_full` 统一出口视图。
2. 建立统一增量刷新编排：基础库刷新、衍生核心表刷新、缓存刷新、视图重建、审计、数据字典刷新。
3. 建立每日质量报告：行数、覆盖率、缺失率、PIT 检查、异常枚举、最新交易日完整性。
4. 将 Phase 3 的运行脚本整合为统一 CLI。
5. 固化比率特殊值公共函数。
6. 建立文档漂移检查，让数据库、schema registry、Excel 字典和主文档保持一致。

## 9. 最终结论

Phase 3 主体模块已完成，可以进入 Phase 4。

本阶段产出的变量库已经具备以下能力：

1. 全 A 股日频主干对齐。
2. 交易、估值、财务、资金、行业、指数、公司行为、治理、综合状态等主要股票分析事实维度覆盖。
3. 核心物理表与完整视图分层。
4. 审计报告和数据字典同步机制。
5. 事实层原则：不生成选股评分、不生成买卖信号、不生成未来收益标签。
